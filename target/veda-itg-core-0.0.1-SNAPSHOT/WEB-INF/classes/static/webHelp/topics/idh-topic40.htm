Use Case 4

 

 

 

Introduction

Workstream

New Entity Creation (NEC)

Use Case Name

Add new fields on NEC screens and create mapping with filing documents and underlying fields

Stakeholder(s)

a.Intertrust entity admin team member

b.Authorized IRIS system user (client or outside counsel)

c.Global Process Alignment team

d.Intertrust Compliance

References

Business requirements discussions with Intertrust

Traceability

NEC_04

 

1.Use Case Description

EDM will provision authorized users :

a.For the government filing document: creating a mapping between a ‘jurisdiction + entity type’ and a government filing document.

b.For relationship KYC : It will also provision authorized users for creating a mapping based on “jurisdiction + service line” with the data and documents required for entity relationship KYC.

c.It will also provision to map a field on the web forms on IRIS / NEC based on “jurisdiction + entity type” and the ultimate filing documents.

d.EDM will also facilitate (authorized Intertrust internal user) to add new fields for a given jurisdiction and entity type to be captured incrementally of the new entity request screens on IRIS portal. EDM. The new field could be either for relationship KYC, creating transaction profile, target contact for identity KYC, changes to operational documents requirements, or due to changes in fields in the government filing forms.

 

[For details on notifications and status updates refer use case NEC_22]

 

1.Actors

Primary Actor – Intertrust entity admin group employee

Secondary Actor – None

1.Triggers

1.Onboarding a new jurisdiction on NEC automation. This includes onboarding both on IRIS and NEC applications.

2.Due to regulatory changes, new filing documents for a given jurisdiction + entity type

3.Due to regulatory changes, new fields on new filing documents or new/edited fields on existing filing documents for a given jurisdiction + entity type

4.Enhanced Intertrust internal compliance standards, requiring incremental information that is required to be collected on IRIS and NEC for entity relationship KYC, dictated by jurisdiction + + global service lines

5.A Change in fields displayed to IRIS for clients and outside counsel users to complete new entity creation service request, dictated by jurisdiction + entity type

6.A Change in fields displayed on EDM screens for Intertrust entity admin users to complete new entity creation service request, dictated by jurisdiction + entity type +service lines

7.Setting up or editing client details to be displayed (data is un-editable and for reference purpose only) on new entity incorporation service request

8.Setting up or editing target contact information to be captured for reaching out to pertinent individuals to collate data and documents for identity KYC

9.Setting up or editing additional data required for creating transaction profile

 

1.Preconditions

1.All possible entity types for a given jurisdiction sourced from ViewPoint and NavOne and available on NEC to choose from is available

2.All possible service lines (Corporate Service, Fund Service, Private Wealth, Capital Markets etc.) for a given jurisdiction sourced from ViewPoint and NavOne and available on NEC to choose from is available

3.None of the mapping functionalities exposed on IRIS, only exposed on NEC

4.Intertrust entity admin user has access to the EDM portal’s mapping utility for a new entity creation to add-and-define/edit/delete fields and map fields

5.Change service request status as per one of the triggers listed above is available to make the changes

6.If the change involves new mapping or change in existing mapping utilizing a field in a government filing document, , agreed by the Compliance Department and updated government filing form is already available in NEC

7.For existing jurisdictions and their underlying entity types, mapping is already available

8.For new jurisdictions, set up is not available and would need to be set up from scratch

9.Client information is already available on IRIS and NEC as this use case covers existing clients ONLY

10.GPA and/or Compliance team requests a change to the admin team.

 

1.Basic Course of Events

1.The first action an admin user takes on the mapping module is to choose a trigger listed in section 4. Triggers above.

2.When the trigger is to onboard a new jurisdiction (Trigger 1), it will need to perform the mappings in the following order:

a.Choose an entity type and for every entity type, map all possible government filing forms

b.Map all fields within a government filing form, with the actual form

c.For every entity type, create field definition for data and documents to be collected from client or outside counsel on IRIS or, from internal Intertrust admin user from NEC. Fields created will be associated with following attributes:

i.Whether for ‘entity relationship KYC’ or for ‘filing form’ or for ‘Both’. For ‘Relationship KYC’ and ‘Both’ fields, ask mandatorily for entity business line

ii.Display name

iii.Additional information / comments to be displayed on the field for explanatory purposes

i❖.Field type – Dropdown single select, dropdown multi-select, free form text, date, phone number, email id, radio button, check box, browse to upload document (for operational documents) etc. For dropdown fields, association to a source data will be required mandatorily

❖.Field is mandatory or not

❖i.To be available for information collection on IRIS, or on NEC, or on both

a.Now, for every entity type, for IRIS and EDM fields that have been setup for ‘filing form’ only (refer #c.(i) above), map every such field with one or more fields of filing forms that need to be filed for that entity type, as per setup in steps (a) and (b) above

b.For every entity type, select from available client related fields to be displayed. Option to have the same client data to be displayed for all entity types within a given jurisdiction should be available

c.For every entity type, to setup fields to capture target contact information for reaching out to pertinent individuals to collate data and documents for identity KYC, create field definition for contact data to be collected. Data could be collected from client or outside counsel on IRIS OR, from entity admin user from NEC. Fields created will be associated with following attributes:

i.Display name

ii.Additional information / comments to be displayed on the field for explanatory purposes

iii.Field type – free form text (for name), date, phone number, email id, radio button, check box etc.

i❖.Mandatory or not

❖.To be available for information collection on IRIS, or on NEC, or on both

a.The data to create transaction profile is the same across all jurisdictions and entity types. Data could be collected from client or outside counsel on IRIS or, from entity admin user from NEC. Fields created will be associated with following attributes:

i.Display name

ii.Additional information / comments to be displayed on the field for explanatory purposes

iii.Field type – free form text (for name), date, phone number, email id, radio button, check box etc.

i❖.Mandatory or not

❖.To be available for information collection on IRIS, or on NEC, or on both

2.Since the steps in #1 are in chronological order, the user should be able to ‘save’ or ‘save and exit’ or ‘go back’ as he/she moves from one step to the next – shall we add the option ‘ cancel’ as well if the client does not want to create an entity anymore ?(sometimes they can just click to test the feature or maybe they realize they need more information).

3.Once mapping is completed, entity admin user on NEC, will save the mapping and exit

 

1.Post Conditions (Basic)

1.Once the mapping is completed and it is time to create a service request, on the choice of jurisdiction+ entity type+ business line, the web form is displayed with four primary categories:

a.Client information (prefilled and read only/not editable)

b.Relationship KYC data

c.Operational documents

d.Target Contact information to collect data and documents to perform identity KYC for new owner-controllers

e.Filing form data

f.Additional data to create transactional profile

1.The fields displayed under each category, both on NEC and IRIS is as per the mapping setup

2.To make any changes, mapping needs to be adjusted depending on the trigger
